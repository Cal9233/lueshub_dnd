Database Schema Update

Important: This page explains how to update your database schema to fix spell saving issues.

The Issue

The issue with spells not saving correctly is due to a missing spells_array_json column in the characters table of your database.

Solution

You need to add this column to your database. You can do this by running the following SQL command in your database management tool:

ALTER TABLE `characters` ADD COLUMN `spells_array_json` LONGTEXT AFTER `known_spells`;
UPDATE `characters` SET `spells_array_json` = '[]' WHERE `spells_array_json` IS NULL;

How to Run This Command

  1. Log into your MySQL database management tool (e.g., phpMyAdmin)
  2. Select the dnd_campaigns database
  3. Go to the SQL tab or query window
  4. Paste the SQL command above
  5. Click Execute or Run

Alternatively: Use the PHP Script

I've also created a PHP script that will make these changes automatically. To use it:

  1. Upload update_schema.php to your server
  2. Visit the URL of this file in your browser (e.g., https://yourdomain.com/update_schema.php)
  3. The script will check if the column exists, and add it if needed

Verify the Fix

After making this change, you should be able to add and save spells properly. The changes will also ensure that existing spells in the legacy format are still accessible.

Note: If you continue to experience issues after making these changes, please check your browser's console for any JavaScript errors and ensure that the spellfix.js script is being loaded correctly.
